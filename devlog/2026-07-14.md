# 2026-07-14 开发日志

## 今日完成

### 飞书账号管理功能 ✅ (端到端打通)

在飞书多维表格新增第 5 张表「账号管理」，支持团队在飞书中直接管理监控账号，无需编辑 YAML 文件。

#### 实现方案

| 文件 | 操作 | 说明 |
|------|------|------|
| `config/bitable_schema.yaml` | 修改 | 新增 `account_manager` 表定义（10 个字段） |
| `src/loaders/account_manager.py` | **新建** | `FeishuAccountManager` — 从飞书加载启用的账号 |
| `src/core/config.py` | 修改 | 新增 `load_accounts_from_bitable()`，`source` 参数支持 yaml/bitable/auto |
| `src/core/pipeline.py` | 修改 | `run_all_accounts()` + `run_pipeline()` 新增 `source` 参数 |
| `src/cli/main.py` | 修改 | `run`/`test-collect` 新增 `--source/-s` 选项；`clear` 新增 `--all` 标志 |
| `src/scheduler/jobs.py` | 修改 | 支持 `ACCOUNT_SOURCE` 环境变量 |

#### 三种账号来源模式

| 模式 | 命令 | 说明 |
|------|------|------|
| `yaml` | `xhs-feishu run` | 从 `accounts.yaml` 读取（缓存） |
| `bitable` | `xhs-feishu run -s bitable` | 从飞书「账号管理」表读取（每次都重新读取） |
| `auto` | `xhs-feishu run -s auto` | 先飞书，失败回退 YAML |

#### 账号管理表字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 账号ID | Text | ✅ | 系统内部唯一标识 |
| XHS用户ID | Text | ✅ | 小红书平台用户 ID |
| XHS用户名 | Text | | 小红书用户名 |
| 显示名称 | Text | | 飞书展示用 |
| 账号类型 | SingleSelect | | 自有账号 / 竞品账号 |
| 启用 | Checkbox | ✅ | 勾选才参与采集 |
| 粉丝数/关注数/总获赞/总收藏 | Number | | 可选基线值 |

#### 过滤规则
- 「启用」Checkbox 未勾选 → 跳过
- 「账号ID」或「XHS用户ID」为空 → 跳过

### 一键操作脚本 ✅

| 文件 | 功能 | 命令 |
|------|------|------|
| `每日采集.bat` | 从飞书读取账号 → 采集同步 | `xhs-feishu run -s bitable` |
| `删除.bat` | 清空 4 张数据表 + SQLite | `xhs-feishu clear --all --confirm` |

#### 修复的问题
20. [x] **bat 文件双击运行失败** — 三个原因：
    - 硬编码 `cd /d` 路径在双击时找不到 → 改用 `cd /d "%~dp0"`（bat 自身目录）
    - `xhs-feishu` 不在双击环境的 PATH 中 → 改用完整路径 `C:\...\Python314\Scripts\xhs-feishu.exe`
    - CMD GBK 编码导致中文乱码 → 全部换为英文
21. [x] **`删除.bat` 双击无效** — 同上 PATH + 编码问题，修复后正常工作

### 用户工作流确认

```
1. 飞书「账号管理」表填写账号信息 → 勾选「启用」
2. 双击「每日采集.bat」 → 自动采集 + 同步
3. 双击「删除.bat」 → 一键清空数据（需输入 YES 确认）
```
