## 晚间 — 浏览器笔记数据自动采集 ✅ (核心突破)

### 目标
消除笔记数据对 CSV 的依赖，实现创作者中心全自动采集。

### API 发现过程
1. 创作者中心首页拦截 5 个 API：`personal_info` / `note_detail_new` / `datacenter/account/base` / `datacenter/livedata/overview` / `datacenter/leaderboard/recommend`
2. `note_detail_new` 返回的是 **7/30 天聚合统计**（非单篇笔记数据），含每日 view/like/collect/comment/share/fans 趋势
3. 启用「API 发现模式」记录全部网络请求，发现 `statistics/data-analysis` 页面调用了：
   - `**/api/galaxy/creator/datacenter/note/analyze/list**` — **单篇笔记互动数据**（关键 API）
   - `/api/galaxy/v2/creator/live_rooms` — 直播数据
4. `note/analyze/list` 响应结构：
   ```json
   {"data": {"total": 1, "note_infos": [{"id": "697c2efa...", "title": "互填",
     "read_count": 57, "like_count": 2, "fav_count": 1,
     "comment_count": 4, "share_count": 0, "post_time": 1769746170000, "type": 1}]}}
   ```

### 解析字段映射
| 创作者中心字段 | 内部模型字段 | 说明 |
|--------------|-------------|------|
| `id` | `note_id` | 非 `note_id` |
| `read_count` | `views` | 浏览量（非 `view_count`）|
| `fav_count` | `favorites` | 收藏（非 `collect_count`）|
| `post_time` | `publish_date` | 发布时间 |
| `type: 1/2` | `note_type` | 1=图文, 2=视频（整数）|

### 合并策略变更
- **旧**: HybridCollector 按优先级降级，谁先返回数据就用谁
- **新**: 浏览器优先 + CSV 补充去重（按 `note_id` 去重）
- 结果：浏览器 1 篇（实时互动数据）+ CSV 补充 21 篇 = 22 篇
