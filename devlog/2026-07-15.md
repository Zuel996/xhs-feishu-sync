### Step 1: PyInstaller 打包 ✅

把 `src/api/server.py` + 所有依赖打包成 `xhs-feishu-server.exe`。

**关键文件**：
- `run_server.py` — PyInstaller 入口，整合系统托盘
- `xhs-feishu-server.spec` — PyInstaller 配置文件（`--onedir --noconsole`）
- `xhs-feishu-server-debug.spec` — 调试版（`--console`）

**输出**：`dist/xhs-feishu-server/`（~111MB），含 `xhs-feishu-server.exe` + `_internal/`

**解决的问题**：
- hidden imports：`src.api`、`src.api.server`、`pystray`、`PIL`
- uvicorn 必须以字符串 `"src.api.server:app"` 方式导入（PyInstaller 限制）
- noconsole 版双击静默运行，系统托盘图标出现

### Step 2: 系统托盘 ✅

**技术方案**：`pystray` + `Pillow`，uvicorn 在 daemon 线程运行，主线程消息泵。

```
┌─────────────────────────┐
│  🔴 xhs-feishu-sync    │
│  ───────────────────── │
│  📊 查看状态            │
│  ❌ 退出                │
└─────────────────────────┘
```

**关键实现** (`run_server.py`)：
- `create_icon_image()` — 64×64 红色圆角矩形 + 白色"S"字
- `start_tray()` — main thread 跑 pystray，daemon thread 跑 uvicorn
- 点击"查看状态" → 打开 `http://localhost:9527/health`

**依赖变更**：`pyproject.toml` 新增 `pystray>=0.19`、`Pillow>=10.0`

### Step 5: 定时自动采集 ✅

Service Worker `chrome.alarms` 每日定时：
- `onInstalled` 时计算距离下次采集的分钟数
- 默认每日 10:00，延迟到毫秒精度
- `onStartup` 更新定时信息
- 采集结果写入 `chrome.storage.local`（`lastCollectResult` + `scheduleInfo`）
- Popup 打开时优先从 storage 读取（无需请求后端）

## 遇到的问题

1. **PyInstaller noconsole exe 静默崩溃** — `start ""` 方式启动无输出。通过直接调用 exe 确认正常运行。
2. **Port 9527 占用** — 多次测试导致端口冲突，`taskkill /f` + `netstat -ano | findstr 9527` 解决。
3. **Edit 工具中文编码问题** — 编辑含中文的 HTML 时字符串匹配失败，改用简短唯一字符串匹配解决。
4. **`cmd /c scripts\package.bat` 在 Git Bash 失败** — 改用 PowerShell `robocopy` + `Compress-Archive` 替代。
