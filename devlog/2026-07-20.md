### Bug 1: Service Worker postMessage Hook 延迟安装失败 🔧

**文件**: `extension/content/xhs-interceptor.js`

**现象**: 当 XHS 页面的 Service Worker 延迟激活时（Content Script 注入时 SW controller 尚未就绪），延迟轮询检测到 SW controller 可用后，仅设置 `DIAG.swController = true`，但从未调用 `installPostMessageHook()` 安装实际的拦截逻辑。

**根因**: `hookSWPostMessage()` 中的 `setInterval` 回调缺少对 `installPostMessageHook()` 的调用。

**修复**:
- 提取独立的 `installPostMessageHook(label)` 函数，将 SW postMessage 拦截逻辑封装其中
- Content Script 加载时立即调用一次 `installPostMessageHook('immediate')`
- 延迟轮询检测到 SW controller 就绪后，通过 `swPostMessageHooked` flag 协调，再次调用 `installPostMessageHook('delayed')`
- 避免重复安装

### Bug 2: `extractProfile()` 守卫条件拼写错误 🔧

**文件**: `extension/content/xhs-interceptor.js`

**现象**: `extractProfile()` 函数的守卫条件中 `result.red_num` 应该是 `result.red_id`。

**修复**: 
```javascript
// Before
if (!result.red_num && !result.name && !result.fans_count)

// After
if (!result.red_id && !result.name && !result.fans_count && !result.fansCount)
```
同时增加了 `fansCount` 驼峰命名变体的兼容检查。

### 架构决策：Chrome Extension → CDP + 轻量前端 🔄

经过两轮讨论，决定**废弃 Chrome Extension 的数据拦截层**，回归更简单、已验证的 CDP 架构。

**决策理由**:
- Chrome Extension 架构过度复杂（Page↔SW↔Popup 三层通信、注入时机竞争、Manifest V3 限制）
- Content Script API 拦截在 XHS 这种重度使用 Service Worker 的 SPA 中极不可靠
- CDP（Chrome DevTools Protocol）采集链路 **已经验证可用**（`XHSBrowserCollector` 在 Phase 2-6 通过了所有测试）
- CDP 走的是浏览器 Network 层拦截，完全不受页面 JS 环境影响

**新架构**:
```
Chrome CDP (:9222)  ←──  Python API Server (:9527)  ←──  Web UI (http://localhost:9527)
     │                        │                              │
     │  Network.getResponseBody  │  POST /collect               │  飞书配置 + 账号管理
     │  (raw WebSocket)         │  内部调用 CDP 采集器          │  一键采集按钮
     │                          │  → Pipeline → Feishu          │  采集状态展示
```

**实施计划**: 已写入 `warm-bubbling-valley.md`，分 3 个 Phase：
| Phase | 内容 | 状态 |
|-------|------|------|
| Phase 1 | `server.py` 改造（/accounts CRUD、/collect 改 CDP 模式、APScheduler、/diagnostics） | ⬜ 待实施 |
| Phase 2 | `webui/` 独立前端（从 extension/popup/ 改造，移除 Chrome API） | ⬜ 待实施 |
| Phase 3 | 启动流程优化 + 端到端验证 | ⬜ 待实施 |

**影响范围**:
- `src/api/server.py` — 主要改动
- `webui/` — 新增目录（3 个文件）
- `extension/` — 废弃但不删除
- 其他模块（collectors、pipeline、sync_engine、storage、transformers）— **完全不动**
