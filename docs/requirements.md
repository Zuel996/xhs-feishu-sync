# 项目需求文档

## 项目名称
小红书 → 飞书多维表格 数据自动统计工具 (xiaohongshu-feishu-sync)

## 项目目标
自动采集公司小红书账号（及竞品账号）的数据指标，经过清洗、计算趋势和排名后，自动同步到飞书多维表格中，实现数据日报自动化。

## 背景
团队需要每天手动查看小红书后台数据并录入飞书表格，效率低且容易出错。

## 功能需求

### F1: 数据采集
- **F1.1** 自动采集小红书账号概览数据：粉丝数、关注数、获赞与收藏总数
- **F1.2** 自动采集笔记互动数据：浏览量、点赞数、收藏数、评论数、分享数
- **F1.3** 支持浏览器自动化采集（CDP模式连接真实Chrome）
- **F1.4** 支持 CSV 文件导入（降级方案）
- **F1.5** 支持小红书开放平台 API 采集（未来，需企业认证）
- **F1.6** 支持自有账号 + 竞品账号分类管理

### F2: 数据转换
- **F2.1** 中文数字解析（"1.2万" → 12000）
- **F2.2** 日环比（DoD）增量计算
- **F2.3** 周环比（WoW）增量计算
- **F2.4** 增长率计算
- **F2.5** 异常检测（3σ 阈值）
- **F2.6** 竞品排名与横向对比

### F3: 数据同步
- **F3.1** 自动同步到飞书多维表格
- **F3.2** 增量同步（仅写变更数据）
- **F3.3** 批量写入（500条/批）
- **F3.4** 幂等性保证（重复运行不产生脏数据）
- **F3.5** 4张数据表：账号概览、笔记明细、每日快照、竞品对比

### F4: 定时调度
- **F4.1** 每日定时自动执行（默认 8:00 AM）
- **F4.2** 失败重试（3次，间隔5分钟）
- **F4.3** 手动触发执行

### F5: 通知推送
- **F5.1** 飞书 Bot 日报推送（成功时）
- **F5.2** 飞书 Bot 告警推送（失败时）

### F6: 本地存储
- **F6.1** SQLite 存储历史快照（90天保留）
- **F6.2** 用于趋势计算和增量 diff 基准

## 非功能需求
- **NF1**: 小红书每日请求量 < 100次/账号（避免反爬）
- **NF2**: 操作间隔 1.5-5s 随机延迟
- **NF3**: 单账号失败不影响其他账号
- **NF4**: Python ≥ 3.11
- **NF5**: Windows 系统兼容

## 约束条件
- 暂无小红书开放平台 API 权限（企业认证 + 蓝V 认证所需）
- Chrome 浏览器 + CDP 调试模式为必要运行环境
- 飞书自建应用 + Bitable 权限为必要配置
